## 概述

Approval Kit 提供了 25 个完整的使用场景示例,从简单到复杂,帮助开发者理解如何使用本库构建审批流程.

## 基础场景

### 场景 1: 最简单的单人审批流程

请假申请,只需要直接主管审批即可.

**特性**: 固定审批人、单人审批模式

### 场景 2: 多人会签审批流程

采购申请,需要财务、采购、总经理全部同意才能通过.

**特性**: 多人会签模式、所有审批人必须全部同意

### 场景 3: 条件分支审批流程

费用报销,根据金额大小选择不同的审批路径.

**特性**: 条件节点、数值比较、动态路径选择

## 高级场景

### 场景 4: 动态审批人获取

项目立项审批,审批人需要根据项目类型和部门动态获取.

**特性**: HTTP API 动态获取、参数映射、响应解析

### 场景 5: 顺序审批流程

员工转正审批,需要按顺序经过直属主管、部门经理、HR、总经理审批.

**特性**: 顺序审批模式、多级审批链

### 场景 6: 比例会签审批流程

技术方案评审,5 位专家中需要 3 位同意即可通过.

**特性**: 比例会签模式、阈值配置

## 完整示例

所有示例代码位于 `examples/` 目录,每个场景包含:

- **main.go**: 完整的可运行代码
- **README.md**: 详细的场景说明和使用指南

## 运行示例

```bash
# 进入场景目录
cd examples/01-single-approval

# 运行示例
go run main.go
```
