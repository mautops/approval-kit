# 场景 07: 拒绝后跳转流程

## 场景描述

这是一个拒绝后跳转流程示例,模拟合同审批场景.当财务审批拒绝时,可以回退到业务部门重新修改.

**业务背景**: 合同审批流程需要经过业务部门和财务审批.如果财务审批拒绝,根据业务规则,应该回退到业务部门重新修改合同,而不是直接终止流程.这种模式适用于需要支持驳回修改的场景.

## 流程结构

```
开始节点 → 业务部门审批 → 财务审批 → 结束节点
```

流程说明:
1. **开始节点**: 流程的起点,自动执行
2. **业务部门审批节点**: 业务部门审批合同
   - 审批模式: 单人审批 (ApprovalModeSingle)
   - 审批人: 业务部门审批人
3. **财务审批节点**: 财务审批合同
   - 审批模式: 单人审批 (ApprovalModeSingle)
   - 审批人: 财务审批人
   - **拒绝后行为**: 回退到上一节点 (RejectBehaviorRollback)
4. **结束节点**: 流程的终点

## 关键特性

- **拒绝后回退**: 使用 RejectBehaviorRollback 实现拒绝后回退到上一节点
- **流程继续**: 拒绝后流程继续,而不是终止
- **重新审批**: 支持业务部门根据财务意见修改后重新审批
- **配置灵活**: 支持配置拒绝后行为(终止/回退/跳转)

## 代码说明

### 主要组件

1. **模板创建** (`createTemplate`):
   - 创建包含业务部门和财务审批节点的模板
   - 配置财务审批节点的拒绝后行为为回退 (RejectBehaviorRollback)
   - 配置节点间的连接关系

2. **任务创建** (`main`):
   - 创建合同审批任务
   - 传入任务参数(合同编号、合同名称、金额、双方信息等)

3. **流程执行** (`runScenario`):
   - 提交任务进入审批流程
   - 设置审批人列表
   - 业务部门审批
   - 说明拒绝后回退的完整流程

4. **结果输出** (`printResults`):
   - 输出任务状态、审批记录
   - 验证拒绝后回退配置

### 代码结构

```go
main()
  ├── 创建管理器 (TemplateManager, TaskManager)
  ├── 创建模板 (createTemplate)
  │   └── 配置财务审批节点 (RejectBehaviorRollback)
  ├── 创建任务 (Create)
  ├── 执行流程 (runScenario)
  │   ├── 提交任务 (Submit)
  │   ├── 设置审批人列表 (AddApprover)
  │   ├── 业务部门审批 (Approve)
  │   └── 说明拒绝后回退流程
  └── 输出结果 (printResults)
```

### 拒绝后回退配置示例

```go
Config: &node.ApprovalNodeConfig{
    Mode: node.ApprovalModeSingle,
    ApproverConfig: &node.FixedApproverConfig{
        Approvers: []string{"finance-001"},
    },
    RejectBehavior: node.RejectBehaviorRollback, // 拒绝后回退到上一节点
}
```

## 适用场景

这个场景适用于以下实际业务场景:

- **合同审批**: 财务拒绝后回退到业务部门重新修改
- **方案审批**: 上级审批拒绝后回退到下级重新修改
- **其他审批流程**: 需要支持驳回修改的场景

## 扩展说明

如果需要扩展此场景,可以考虑:

1. **测试拒绝后跳转**: 配置 RejectBehaviorJump,拒绝后跳转到指定节点
2. **测试拒绝后终止**: 配置 RejectBehaviorTerminate,拒绝后终止流程
3. **动态审批人**: 结合动态审批人配置,根据合同信息动态确定审批人
4. **条件分支**: 根据合同金额或类型,选择不同的审批路径
5. **审批超时**: 配置审批超时时间,超时后自动处理
